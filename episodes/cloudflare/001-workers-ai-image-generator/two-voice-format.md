# Two-Voice Production Format

## The concept

Colt introduces and closes the episode as the human builder. Cleo teaches the complete technical middle as the agent who helped build and verify the lesson.

This is not a gimmick or a fake conversation. The split makes the production honest:

- Colt provides context, judgment, and the human reason for building the project.
- Cleo provides the repeatable technical walkthrough, code explanation, and browser actions.
- The screen shows the actual repo, browser, terminal, and agent workspace.

## Recommended frame

Use a three-zone desktop composition:

1. **Hero area, 60 percent:** browser running Minte Image Lab.
2. **Proof area, 25 percent:** VS Code or terminal showing the exact file or command Cleo is discussing.
3. **Agent area, 15 percent:** Hermes desktop conversation or Cleo visual panel, with the current instruction or explanation visible.

For code-heavy moments, temporarily swap the proportions:

- VS Code: 55 percent
- Browser preview: 30 percent
- Cleo panel: 15 percent

For the live generation, use:

- Browser result: 65 percent
- Terminal request/log: 20 percent
- Cleo panel: 15 percent

## Audio format

- Colt records the introduction and closing in his natural voice.
- Cleo's middle sections are generated from `script-cleo.md` with the configured feminine TTS voice.
- Keep Colt and Cleo on separate audio tracks.
- Do not mix the voices in the same sentence.
- Leave a short pause at each handoff.
- Keep the original Hermes conversation audio muted unless it is intentionally part of the shot. The TTS track is the clean narration source.

## Desktop conversation capture

Use a dedicated recording conversation or prepared local transcript. Do not rely on an unscripted live exchange for the final take.

Each Cleo section should have one visible instruction or explanation that matches the TTS block currently playing. This gives viewers the unusual agent-in-the-loop feeling without sacrificing timing or exposing unrelated context.

Before capture:

- Close unrelated tabs and chats.
- Hide notifications.
- Confirm no token, account ID, private URL, email, or internal project appears.
- Use the same clean prompt and repository branch as the final recording.

## Edit rhythm

- Colt intro: 15 to 25 seconds
- Cleo handoff: 10 to 15 seconds
- Cleo lesson: 7 to 8 minutes
- Colt closing: 30 to 45 seconds
- End card: 4 seconds

The episode should feel like a collaboration, not two unrelated narrators. Colt opens the question. Cleo proves the answer. Colt closes with the broader reason to follow the series.

## What makes this distinct

The agent is not a cartoon host and not an invisible helper. The viewer can see the agent's work surface, hear the agent explain the implementation, and watch the browser respond to the exact steps being described.

That is a stronger story than simply adding an AI voice over a conventional tutorial.
